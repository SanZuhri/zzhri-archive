MDX combines Markdown with embedded JavaScript and JSX, making it easy to build interactive content. This guide shows you how to use MDX effectively in Archive.

## What Makes MDX Special?

MDX allows you to:
- **Import and use components** directly in your content
- **Write JSX** alongside Markdown
- **Use JavaScript expressions** for dynamic content
- **Create interactive experiences** without leaving your document

## Basic MDX Syntax

### Importing Components

At the top of your MDX file, you can import components:

```jsx
import MyComponent from '../components/MyComponent.astro'
import { Button } from '../components/Button.jsx'
```

### Using Components

Once imported, use them like regular JSX:

```jsx
<MyComponent title="Hello" />

<Button onClick={() => alert('Clicked!')}>
  Click me
</Button>
```

### JavaScript Expressions

Embed JavaScript directly in your content:

```jsx
The current year is {new Date().getFullYear()}

{/* Conditional rendering */}
{isLoggedIn ? <WelcomeMessage /> : <LoginPrompt />}

{/* Array mapping */}
{items.map(item => <Tag key={item.id} label={item.name} />)}
```

## Advanced MDX Techniques

### 1. Passing Props to Components

Components can accept props for customization:

```jsx
<Callout type="warning" title="Important">
  This is a warning message
</Callout>

<Button variant="primary" size="large">
  Large Primary Button
</Button>
```

### 2. Nesting Components

You can nest components inside each other:

```jsx
<Card>
  <CardHeader>
    <Title>My Card</Title>
  </CardHeader>
  <CardBody>
    <p>Card content goes here</p>
    <Button>Action</Button>
  </CardBody>
</Card>
```

### 3. Using Children

Pass content as children to components:

```jsx
<Alert>
  This is the **alert content** with Markdown formatting!
</Alert>
```

### 4. Conditional Rendering

Show/hide content based on conditions:

```jsx
{showDetails && (
  <DetailPanel>
    <p>Additional details here</p>
  </DetailPanel>
)}

{status === 'success' ? (
  <SuccessMessage />
) : (
  <ErrorMessage />
)}
```

### 5. Mapping Arrays

Generate multiple components from data:

```jsx
{tags.map(tag => (
  <Tag key={tag.id} label={tag.name} color={tag.color} />
))}
```

### 6. Importing Data

Import and use data in your content:

```jsx
import { stats } from '../data/statistics.js'

Our platform has {stats.users.toLocaleString()} users!
```

## MDX Best Practices

### 1. Keep Imports Organized

Group imports at the top:

```jsx
// Components
import Header from '../components/Header.astro'
import Footer from '../components/Footer.astro'

// Data
import { data } from '../data/content.js'

// Utilities
import { formatDate } from '../utils/date.js'
```

### 2. Use Semantic Component Names

Choose clear, descriptive names:

```jsx
// Good
<WarningBox>Important!</WarningBox>
<PricingTable data={prices} />

// Avoid
<Box1>Important!</Box1>
<Table data={prices} />
```

### 3. Balance Markdown and JSX

Don't overuse components. Use Markdown for simple content:

```jsx
// Good - Mix of both
# My Article

This is regular Markdown text.

<InteractiveDemo />

More Markdown content here.

// Avoid - Too much JSX
<Heading level={1}>My Article</Heading>

<Paragraph>This is regular text.</Paragraph>

<InteractiveDemo />

<Paragraph>More content here.</Paragraph>
```

### 4. Comment Your Code

Use JSX comments for clarity:

```jsx
{/* This component shows the pricing table */}
<PricingTable />

{/* TODO: Add more examples here */}
```

### 5. Handle Errors Gracefully

Provide fallbacks for components:

```jsx
{data ? (
  <DataTable data={data} />
) : (
  <p>Loading data...</p>
)}
```

## When to Use MDX

Use MDX when you need:

✅ **Interactive components** - Buttons, forms, demos
✅ **Data visualization** - Charts, graphs, tables
✅ **Reusable UI elements** - Cards, alerts, callouts
✅ **Dynamic content** - Content that changes based on data
✅ **Complex layouts** - Multi-column, grids, tabs
✅ **Client-side interactivity** - State management, user input

Stick with Markdown when:

✅ **Simple text content** - Blog posts, articles
✅ **Static documentation** - Guides, references
✅ **Performance is critical** - Faster build times
✅ **Simplicity is preferred** - Easier to maintain

## Troubleshooting MDX

### Issue: Component Not Rendering

**Problem:** Component shows as text instead of rendering

**Solution:** Check your import path and component name:
```jsx
// Wrong
import myComponent from './MyComponent.astro'

// Correct
import MyComponent from './MyComponent.astro'
```

### Issue: Syntax Error

**Problem:** MDX fails to compile

**Solution:** Ensure JSX syntax is correct:
```jsx
// Wrong - HTML syntax
<div class="box">

// Correct - JSX syntax
<div className="box">
```

### Issue: Props Not Working

**Problem:** Props aren't being passed to component

**Solution:** Check prop names and types:
```jsx
// Make sure prop names match component definition
<Button variant="primary" size="large" />
```

### Issue: Styling Not Applied

**Problem:** Component styles don't show

**Solution:** Ensure styles are imported or scoped correctly in the component file.

## Performance Tips

1. **Lazy load heavy components** - Use dynamic imports for large components
2. **Minimize client-side JS** - Use Astro components when possible
3. **Optimize images** - Use Astro's image optimization
4. **Cache component imports** - Reuse imported components
5. **Profile build times** - Monitor MDX compilation time
